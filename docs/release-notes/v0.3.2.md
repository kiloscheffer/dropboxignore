## What's fixed

### Daemon no longer crashes on shape-mismatched `state.json`

`state._read_at()` caught only `json.JSONDecodeError` from the JSON parser and called `_decode(raw)` outside the try/except. `_decode` directly indexed nested `last_error` sub-keys, so a hand-edited `state.json` or a state file produced by a different schema version (still valid JSON, but shaped wrong) raised `KeyError`/`TypeError`/`ValueError` from `_decode`. That exception propagated out of `daemon.run`'s singleton check and the daemon crashed on startup.

systemd's `Restart=on-failure RestartSec=60s` would eventually recover (each restart attempts to re-read state and would retry the crash until something rewrites `state.json`), but the recovery is loud-and-slow rather than the silent fall-through-to-fresh-start that the JSON-syntax-error case got.

The fix moves `_decode(raw)` inside the try/except and broadens the catch to `(json.JSONDecodeError, KeyError, TypeError, ValueError)`. Same recovery shape as before â€” log WARNING, return `None`, daemon treats it as "no prior state" and starts fresh.

This is the symmetric pair to the v0.3.1 atomic-write fix: v0.3.1 closed the *write-side* torn-JSON failure mode, v0.3.2 closes the *read-side* shape-mismatch failure mode. Together the two cover both halves of the state-file I/O contract.

## Internal cleanup

### `_classify()` returns root, eliminating a per-event double-lookup

`daemon._classify()` was calling `find_containing(src, roots)` purely as a gate (the return value was discarded), then `_dispatch` called it again to obtain the actual root. Two passes over the roots list per accepted watchdog event. The fix widens `_classify`'s return shape to include the root, so `_dispatch` unpacks it from the classification instead of looking it up a second time.

Per-event work only â€” not in any per-file hot path. No observable behavior change; this is purely a code-cleanup landing as part of the second-look code-review pass.
