First cross-platform release. Linux support alongside the existing Windows port, plus rule-conflict detection for `.dropboxignore` negations that can't translate to Dropbox's inheritance model, and Conventional-Commits CI enforcement.

## What's new

### Linux first-class support

Tested on Ubuntu 22.04+ and other modern Linux distributions with a systemd user session. Uses `user.com.dropbox.ignored` xattrs — the Dropbox-Linux equivalent of Windows' NTFS alternate data streams. `dropboxignore install` writes `~/.config/systemd/user/dropboxignore.service`; state and logs live under `$XDG_STATE_HOME/dropboxignore/` (fallback `~/.local/state/dropboxignore/`). Daemon logs stream to both the rotating file and systemd-journald (`journalctl --user -u dropboxignore.service`).

### Conflict-detected negations

`.dropboxignore` patterns like `build/` + `!build/keep/` can't work the way gitignore users expect — Dropbox's own ignored-folder inheritance makes the negation inert regardless of what xattrs we set. dropboxignore now detects these conflicts at rule-load time, drops the negation from the active rule set, and surfaces the mismatch via:

- A WARNING log on the daemon path.
- A `rule conflicts (N):` section in `dropboxignore status`.
- A `[dropped]` annotation in `dropboxignore explain`, pointing at the masking rule.

Negations that don't conflict with an ignored ancestor (`*.log` + `!important.log`) still work normally.

### Escape hatches

`DROPBOXIGNORE_ROOT` environment variable overrides Dropbox's own root discovery for non-stock installs. Automatically forwarded into the generated systemd unit at `dropboxignore install` time so shell-exported values survive the service boundary.

### CI & repo hygiene

- Conventional Commits + Conventional Branch enforcement via [`commit-check-action`](https://github.com/commit-check/commit-check-action) on every PR.
- Linux test leg runs on `ubuntu-latest` alongside the existing Windows leg.

## Breaking changes

Two items worth a script audit:

- **`dropboxignore uninstall --purge`** now also deletes `state.json`, `daemon.log*`, the state directory (if empty), and the systemd drop-in directory on Linux — not just ignore markers. Dropbox's sync behavior is unchanged; only local dropboxignore bookkeeping is removed.
- **`dropboxignore explain`** output format changed to match `dropboxignore status` — compact relative paths, two-space field separators, no `=`/`!=` arrow column. Include/negation distinction is now conveyed by the leading `!` on the raw pattern text.
