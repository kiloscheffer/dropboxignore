First PyPI release under the new name.

**Rename.** The project was previously `dropboxignore`. That PyPI distribution name is taken by an unrelated 2019 project (MichalKarol/dropboxignore), so v0.3.0 renames the tool to `dbxignore`. Every project-owned surface moves — CLI, daemon binary, Python package, env vars, state + log directory, systemd unit, Windows Task Scheduler entry, PyInstaller outputs, and the GitHub repo. `.dropboxignore` rule files and the `com.dropbox.ignored` marker key (Dropbox's own contract) are unchanged.

## Upgrading from v0.2.x

One-time manual upgrade:

```bash
dropboxignore uninstall --purge   # on v0.2.x — removes state, logs, service
pip install dbxignore              # or: uv pip install dbxignore
dbxignore install                  # registers the new service under new names
```

`.dropboxignore` rule files in your Dropbox tree carry over untouched — they're never modified by install/uninstall.

## Breaking — installation surface changes

| Surface | v0.2.x | v0.3.0 |
|---|---|---|
| `pip install` name | `dropboxignore` | `dbxignore` |
| CLI | `dropboxignore` | `dbxignore` |
| Daemon | `dropboxignored` | `dbxignored` |
| Env vars | `DROPBOXIGNORE_ROOT`, `DROPBOXIGNORE_DEBOUNCE_*`, `DROPBOXIGNORE_LOG_LEVEL` | `DBXIGNORE_ROOT`, `DBXIGNORE_DEBOUNCE_*`, `DBXIGNORE_LOG_LEVEL` |
| systemd unit (Linux) | `dropboxignore.service` | `dbxignore.service` |
| Task Scheduler (Windows) | `dropboxignore` | `dbxignore` |
| State + log dir (Linux) | `$XDG_STATE_HOME/dropboxignore/` | `$XDG_STATE_HOME/dbxignore/` |
| State + log dir (Windows) | `%LOCALAPPDATA%\dropboxignore\` | `%LOCALAPPDATA%\dbxignore\` |
| Binaries | `dropboxignore.exe`, `dropboxignored.exe` | `dbxignore.exe`, `dbxignored.exe` |

## For maintainers: release workflow changes

- First PyPI publish via Trusted Publishing (OIDC) — no PyPI API token stored as a repo secret.
- Release workflow split into `build`, `publish-github`, `publish-pypi` jobs. PyPI publish runs after manual approval on the `pypi` GitHub environment. GitHub Release is unchanged (tag-gated, authored under `GH_RELEASE_TOKEN` PAT owner when configured).
