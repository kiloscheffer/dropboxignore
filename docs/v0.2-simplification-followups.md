# v0.2 simplification follow-ups

Code-level simplification candidates surfaced during the 2026-04-20 `/simplify` pass on the `feat/v0.1` branch. All tier-1 items (conftest extraction, `roots.find_containing`, daemon tidy-ups, in-place-rename reconcile skip) were applied in that pass. The items below were deliberately deferred — they're design-change territory, not pure simplification, and each deserves its own PR with focused tests.

## Deferred items

### 1. Collapse `RuleCache`'s three parallel dicts

**Where:** `src/dropboxignore/rules.py` — `_specs`, `_lines`, `_pattern_entries`.

**Why deferred:** `match()` walks `_specs[ignore_file].patterns` while `explain()` walks `_pattern_entries[ignore_file]`. The two data shapes carry overlapping information but different invariants (pathspec-version-robustness in `_pattern_entries`). Merging them touches the hottest code path and risks shifting rule-precedence semantics.

**Shape of fix:** single `dict[Path, _LoadedRules]` dataclass holding `lines` + `entries`; `match()` iterates `entries` the same way `explain()` does.

### 2. Use `PathSpec.match_file()` instead of manual `spec.patterns` iteration

**Where:** `rules.RuleCache.match()` loops over `spec.patterns` in Python and calls `pattern.match_file(rel_str)` per pattern, losing pathspec's internal short-circuit optimizations.

**Why deferred:** changes the decision surface for overlapping positive/negative patterns. Needs tests covering: last-match-wins across ancestors, `!` negations at the same precedence level, and pathspec's "include=None" null-op semantics.

**Shape of fix:** call `spec.match_file(rel_str)` per ancestor, combine results with the current "deeper-wins" rule at the ancestor granularity.

### 3. One-pass pattern-entry build in `_load_file`

**Where:** `rules.RuleCache._load_file` calls `_build_spec([raw])` per non-blank/non-comment line just to recover a single `Pattern`.

**Why deferred:** the current per-line reparse is explicitly defensive against pathspec-version differences in how null-op entries are preserved in `spec.patterns` (see inline comment at `rules.py` load-file). Replacing it needs a version-compat test matrix.

**Shape of fix:** walk `spec.patterns` once, map by order to non-blank/non-comment source-line indices, fall back to single-line reparse only on a detected mismatch.

### 6. Condition-variable debouncer instead of 20 ms polling

**Where:** `debounce.Debouncer._run` polls `self._pending` every 20 ms regardless of queue state.

**Why deferred:** Current implementation is correct and well-tested. Switching to `threading.Condition` changes the thread-safety model; needs an updated stress test (especially around `submit`-while-emitting).

**Shape of fix:** worker waits on a `Condition` with timeout = soonest pending deadline; `submit` notifies when it enters a sooner deadline than the current wait.

### 7. Skip reloading unchanged `.dropboxignore` files on sweep

**Where:** `daemon._sweep_once` calls `cache.load_root(r)` every hour, which re-reads and re-parses every `.dropboxignore` under the root regardless of whether watchdog events already kept the cache current.

**Why deferred:** needs mtime/size bookkeeping on the cache to decide "unchanged"; also raises a correctness question about what happens if watchdog missed an event (the sweep is the safety net).

**Shape of fix:** `RuleCache.load_root(root, force=False)` that stats each ignore file and skips unchanged ones; keep the full-reload path for startup or `force=True`.

## Non-deferrals (deliberately skipped as findings)

- **`cli.ads` vs `reconcile.ads` monkeypatch:** the `raising=False` in the pre-refactor `fake_ads` fixture was a no-op belt-and-suspenders — `cli` does import `ads` as a module attribute. The unified conftest fixture no longer needs it.
- **Parallel root reconcile:** independent roots could reconcile in parallel, but typical setups have 1–2 roots and Windows ADS writes serialize per-file anyway. Not worth the hazard.
